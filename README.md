# Git Lab Assignment

## What I Learned
- How to initialize a Git repository and track file changes
- How to connect a local repo to GitHub and push code
- The difference between staged and unstaged changes
- How branching allows parallel development without conflicts
- How to create Pull Requests and collaborate via GitHub
- How to identify and resolve merge conflicts manually

## Challenges Faced
- Understanding the staging area concept at first
- Remembering to switch branches before making changes
- Resolving merge conflicts felt confusing initially, but the markers helped

## How Git Helps in Real Projects
Git enables teams to work on the same codebase simultaneously without overwriting each other's work. Branching keeps features isolated, pull requests enforce code review, and commit history provides a full audit trail. If something breaks, you can roll back to any previous commit instantly.
